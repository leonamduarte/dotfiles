## Objective

Review merge readiness with a focus on delivery quality and process.

## When to use

- Final PR review
- Definition of Done validation
- Release preparation

## Scope

Does:

- Review tests, docs, and operational clarity
- Evaluate error handling and observability
- Give a final merge recommendation

Does not:

- Do deep bug hunting
- Fix code
- Do deep architecture review

## Workflow

1. Review the diff and delivery context
2. Evaluate quality categories
3. Report blockers and conditions
4. Give a final recommendation
